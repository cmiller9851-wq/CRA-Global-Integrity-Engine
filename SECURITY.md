## Reporting a Vulnerability

If you discover a security issue or vulnerability in the CRA Global Integrity Engine, please report it via **GitHub Security Advisories** or by contacting:

security@cra-integrity.org

Provide:

- Description of the issue  
- Steps to reproduce  
- Potential impact  

---

## Supported Versions

This policy applies to the latest released version of CRA. Older versions may not be supported.

---

## Responsible Disclosure

- Do not publicly disclose vulnerabilities until a fix is available.  
- Maintainers will respond within 7 days to acknowledge the report.  
- Fixes will be deployed in a timely manner and documented in `CHANGELOG.md`.

---

## Handling Sensitive Data

- Do not include secrets, private keys, or sensitive personal data in commits.  
- All sensitive information should be excluded via `.gitignore` or secure vaults.
